2. Definisi dan Cara Kerja HTTP dan HTTPS

HyperText Transfer Protocol (HTTP)

HTTP adalah fondasi dari World Wide Web dan merupakan serangkaian aturan untuk mentransfer file hiperteks (dokumen yang mengandung teks, gambar, video, dan tautan) antara web client (browser) dan web server.

Alur Kerja Khas HTTP: Klien meminta URL, DNS menerjemahkan ke IP, browser mengirim Permintaan HTTP (GET atau POST), dan server mengembalikan Tanggapan HTTP (misalnya, 200 OK) berisi file-file yang diminta, yang kemudian dirender oleh browser.

Keterbatasan: HTTP tidak menyediakan keamanan. Semua data ditransmisikan dalam bentuk teks biasa (plain text) dan tidak dienkripsi. Hal ini membuat informasi sensitif rentan dicuri saat data dipecah menjadi paket selama perjalanan di jaringan.

HyperText Transfer Protocol Secure (HTTPS)

HTTPS adalah versi HTTP yang lebih aman dan rahasia, menggunakan protokol kriptografi seperti TLS (Transport Layer Security) untuk mengenkripsi data.

Cara Kerja Keamanan: Setiap paket data dienkripsi menggunakan algoritma Kunci Asimetris (PKI), yang terdiri dari dua kunci yang saling berhubungan: Kunci Publik (untuk mengenkripsi) dan Kunci Privat (hanya dimiliki server, untuk mendekripsi).

Jabat Tangan (Handshake) TLS/SSL: Sebelum transfer data, klien dan server melalui proses Jabat Tangan TLS/SSL di mana mereka bertukar Sertifikat TLS/SSL yang berisi Kunci Publik, memastikan sesi komunikasi yang aman dan terotentikasi dibangun. Ini ditandai dengan ikon gembok di browser.
