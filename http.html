Definisi dan Cara Kerja HTTP dan HTTPS

1. Definisi Hypertext Transfer Protocol (HTTP)

HTTP (Hypertext Transfer Protocol) adalah protokol lapisan aplikasi (lapisan tertinggi dalam model OSI) yang menjadi tulang punggung dari World Wide Web (WWW).

Secara sederhana, HTTP adalah seperangkat aturan yang menentukan bagaimana Client (yaitu browser Anda) dan Server berkomunikasi untuk mentransfer hypertext (konten web seperti teks, gambar, atau video).

  • Sifat Dasar: HTTP bersifat stateless, artinya server tidak menyimpan informasi atau status interaksi sebelumnya dari klien.
  • Port Standar: Secara default, HTTP menggunakan Port 80.
  • Keamanan: Data yang ditransfer melalui HTTP dikirimkan dalam bentuk plaintext (teks biasa) tanpa enkripsi. Hal ini membuat data rentan disadap dan dibaca oleh pihak ketiga saat transit melalui jaringan.

2. Cara Kerja HTTP

Cara kerja HTTP mengikuti model permintaan-respons (request-response) sederhana:

  1. Koneksi: Klien (browser) membuat koneksi TCP/IP ke Server (biasanya melalui Port 80).
  2. Permintaan (Request): Klien mengirimkan pesan permintaan HTTP (misalnya, HTTP GET untuk meminta halaman) ke Server.
  3. Respons (Response): Server memproses permintaan tersebut.
  4. Pengiriman Data: Server mengirimkan data yang diminta (misalnya, file HTML) kembali ke klien, disertai dengan Kode Status (misalnya, 200 OK yang berarti berhasil).
  5. Penutupan Koneksi: Server kemudian menutup sambungan TCP/IP.

Ilustrasi browser dan server terhubung: di HTTP, data digambarkan teks terbuka (plaintext); di sisi HTTPS, data digambarkan sebagai teks acak (terenkripsi) di dalam tunnel aman.

3. Definisi Hypertext Transfer Protocol Secure (HTTPS)

HTTPS (Hypertext Transfer Protocol Secure) adalah perpanjangan dari HTTP yang menambahkan lapisan keamanan. Huruf 'S' pada HTTPS berarti Secure (Aman).

HTTPS mengatasi kelemahan utama HTTP dengan menggunakan protokol keamanan SSL (Secure Sockets Layer) atau penerusnya, TLS (Transport Layer Security), untuk mengenkripsi data yang ditransfer antara browser dan server.

  • Port Standar: HTTPS menggunakan Port 443.
  • Keamanan: Semua data dikirimkan dalam bentuk terenkripsi, menjadikannya tidak dapat dibaca oleh penyadap, sehingga melindungi informasi sensitif (seperti detail login atau kartu kredit) dari serangan Man-in-the-Middle (MITM).

4. Cara Kerja HTTPS (Peran SSL/TLS)

Proses HTTPS sedikit lebih kompleks karena melibatkan verifikasi dan enkripsi, yang dikenal sebagai TLS Handshake:

  1. Permintaan Sertifikat: Saat Anda mengunjungi situs HTTPS, browser Anda meminta Server untuk mengidentifikasi dirinya.
  2. Verifikasi (Handshake): Server merespons dengan mengirimkan Sertifikat SSL/TLS yang berisi Kunci Publik server. Browser kemudian memverifikasi sertifikat ini dengan otoritas sertifikasi.
  3. Enkripsi Sesi: Setelah verifikasi berhasil, browser dan server menggunakan Kunci Publik dan Kunci Privat untuk menyepakati Kunci Sesi yang unik. Kunci sesi inilah yang akan digunakan untuk mengenkripsi dan mendekripsi semua data yang dipertukarkan selama sesi tersebut.
  4. Transfer Data: Barulah data permintaan dan respons (yang kini terenkripsi) dikirim melalui koneksi aman.
