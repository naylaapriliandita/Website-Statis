Definisi dan Cara Kerja Nama Domain

1. Definisi dan Struktur Nama Domain

Nama Domain adalah alamat unik yang mudah diingat yang digunakan untuk mengidentifikasi suatu situs web atau server di Internet. Domain berfungsi sebagai pengganti bagi Alamat IP numerik yang rumit (misalnya, 104.26.4.103).

  • Fungsi Utama: Domain diciptakan untuk memudahkan pengguna internet mengakses situs web tanpa perlu menghafal kombinasi angka Alamat IP yang panjang.
  • Struktur Hierarki: Setiap domain terdiri dari beberapa bagian yang memiliki tingkatan:
    • TLD (Top-Level Domain): Bagian paling kanan atau ekstensi (e.g., .com, .org, .id).
    • SLD (Second-Level Domain): Nama yang Anda pilih (nama brand atau situs Anda).
    • Subdomain: Domain di bawah domain utama yang berdiri sendiri (e.g., blog pada blog.namadomain.com).

Anatomi Nama Domain (Subdomain + Domain + TLD), yang disebut dengan Fully Qualified Domain Name (FQDN).

2. Cara Kerja Domain (Keterkaitan dengan DNS)

Domain hanyalah label; agar berfungsi, ia harus diterjemahkan menjadi alamat numerik oleh DNS:

  1. Input Domain: Anda mengetikkan nama domain (contoh.com) di browser.
  2. Terjemahan (DNS Lookup): Browser mengirimkan permintaan ke DNS Server untuk mencari Alamat IP yang terikat pada domain tersebut.
  3. Pengarahan ke Server: Alamat IP tersebut digunakan komputer untuk menuju server dan meminta data website.
  4. Koneksi Server: Setelah Alamat IP ditemukan, browser Anda menggunakan alamat numerik tersebut untuk membuat koneksi langsung ke server yang menyimpan data situs web.
